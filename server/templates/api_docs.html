🔐 认证方式

所有 /api/v1/ 接口均需要 API Key 认证。认证方式有两种：

方式一：请求头（推荐）

X-API-Key: dc_your_api_key_here

方式二：查询参数

GET /api/v1/devices?api_key=dc_your_api_key_here
⚠️ API Key 相当于密码，请勿泄露。建议通过请求头传递，避免在 URL 中暴露。

权限说明

权限说明可访问的接口
read只读所有 GET 请求
write读写所有请求（包含 POST/PUT/DELETE）

在 管理端 → 🔑 API 管理 中创建和管理 API Key。

📦 响应格式

所有 V1 API 均使用统一的 JSON 响应格式：

{
    "code": 200,
    "message": "success",
    "data": { ... },
    "timestamp": "2026-04-14T10:30:00"
}
字段类型说明
codeintHTTP 状态码
messagestring响应消息
dataobject/array响应数据
timestampstring服务器时间戳

💻 设备管理

GET /api/v1/devices
获取设备列表，支持分页和筛选

查询参数

参数类型必填说明
project_idint否按项目ID筛选
department_idint否按单位ID筛选
keywordstring否关键词搜索（姓名/电脑名/IP）
pageint否页码，默认 1
per_pageint否每页数量，默认 20，最大 100

响应示例

{
    "code": 200,
    "message": "success",
    "data": {
        "items": [{ /* 设备对象 */ }],
        "total": 150,
        "page": 1,
        "per_page": 20,
        "total_pages": 8
    },
    "timestamp": "2026-04-14T10:30:00"
}
GET /api/v1/devices/:id
获取单个设备详情

路径参数

参数类型说明
idint设备ID
POST /api/v1/devices
创建设备记录（需要 write 权限）

请求体 (JSON)

字段类型必填说明
department_idint✅所属单位ID
user_namestring✅使用人
user_phonestring联系电话
user_positionstring安装位置
computer_namestring电脑名称
ip_addressstringIP 地址
mac_addressstringMAC 地址
dhcp_enabledstringDHCP (是/否)
os_infostring操作系统
cpu_infostringCPU
ram_infostring内存
disk_infostring硬盘
motherboard_infostring主板
gpu_infostring显卡
network_adapterstring网卡
subnet_maskstring子网掩码
gatewaystring默认网关
dns_serversstringDNS 服务器
forceboolean强制提交（忽略IP/MAC重复），默认 false
⚠️ 如果 IP 或 MAC 与已有设备重复且 force 未设为 true，将返回 409 状态码并提供重复设备详情。
PUT /api/v1/devices/:id
更新设备记录（需要 write 权限）

请求体与创建接口相同，仅传入需要修改的字段即可。

DELETE /api/v1/devices/:id
删除设备记录（需要 write 权限）
GET /api/v1/devices/check-duplicates
检测所有 IP/MAC 地址重复
GET /api/v1/export/:format
导出设备列表

路径参数

参数说明
format导出格式：xlsx 或 csv

查询参数

参数说明
project_id按项目筛选
department_id按单位筛选
keyword关键词搜索

❌ 错误码说明

状态码说明
200请求成功
201创建成功
400请求参数错误
401未认证（缺少或无效的 API Key）
403权限不足
404资源不存在
409数据冲突（IP/MAC重复等）
500服务器内部错误

💡 调用示例

Python (requests)

import requests

API_BASE = "http://localhost:5000/api/v1"
API_KEY = "dc_your_api_key_here"
HEADERS = {"X-API-Key": API_KEY}

# 获取设备列表
resp = requests.get(f"{API_BASE}/devices", headers=HEADERS, params={
    "page": 1,
    "per_page": 10,
    "project_id": 1
})
data = resp.json()
print(f"共 {data['data']['total']} 条设备")

# 创建设备记录
resp = requests.post(f"{API_BASE}/devices", headers={**HEADERS, "Content-Type": "application/json"}, json={
    "department_id": 1,
    "user_name": "张三",
    "computer_name": "PC-001",
    "ip_address": "192.168.1.100",
    "mac_address": "AA:BB:CC:DD:EE:FF"
})
print(resp.json())

# 导出 Excel
resp = requests.get(f"{API_BASE}/export/xlsx", headers=HEADERS)
with open("devices.xlsx", "wb") as f:
    f.write(resp.content)

cURL

# 获取设备列表
curl -H "X-API-Key: dc_your_api_key_here" \
     "http://localhost:5000/api/v1/devices?page=1&per_page=10"

# 创建设备记录
curl -X POST -H "X-API-Key: dc_your_api_key_here" \
     -H "Content-Type: application/json" \
     -d '{"department_id":1,"user_name":"张三","ip_address":"192.168.1.100"}' \
     "http://localhost:5000/api/v1/devices"

# 获取统计信息
curl -H "X-API-Key: dc_your_api_key_here" \
     "http://localhost:5000/api/v1/stats"

JavaScript (fetch)

const API_BASE = 'http://localhost:5000/api/v1';
const API_KEY = 'dc_your_api_key_here';

// 获取设备列表
const res = await fetch(`${API_BASE}/devices?page=1`, {
    headers: { 'X-API-Key': API_KEY }
});
const data = await res.json();
console.log(`共 ${data.data.total} 条设备`);

// 创建设备
const res2 = await fetch(`${API_BASE}/devices`, {
    method: 'POST',
    headers: { 'X-API-Key': API_KEY, 'Content-Type': 'application/json' },
    body: JSON.stringify({
        department_id: 1,
        user_name: '张三',
        ip_address: '192.168.1.100'
    })
});
